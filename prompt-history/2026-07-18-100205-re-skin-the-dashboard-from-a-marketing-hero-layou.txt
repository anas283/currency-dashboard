❯ Re-skin the dashboard from a marketing-hero layout to a dashboard-first       
  layout, keeping DESIGN.md's color/type/spacing/radius tokens but changing     
  structure and component roles:                                                
                                                                                
  1. Header/toolbar: replace the sage hero band + giant display-mega headline   
  with a compact top toolbar (single row, no scroll needed). Include: logo,     
  nav, live/offline status badge, last-updated timestamp, theme toggle — all    
  visible without scrolling.                                                    
                                                                                
  2. Converter: convert from a one-off hero card into a persistent widget —     
  either a fixed sidebar panel or a sticky card — so it stays usable while      
  scrolling the table/chart. Drop the oversized marketing-card treatment; keep  
  it compact and functional.                                                    
                                                                                
  3. Rate table: give it primary real estate (top of content area, not buried   
  below hero). Add:                                                             
     - Sortable column headers (currency code, rate, base) with visible sort    
  affordance.                                                                   
     - Colored rate-change deltas (green up / red down) per row if historical   
  data available.                                                               
     - Sticky search/filter bar above the table.                                
                                                                                
  4. Trend chart: present as its own clear card with the 3-currency selector    
  and daily/weekly/monthly toggle visible together, not a secondary/buried      
  panel.                                                                        
                                                                                
  5. Offline mode: add an explicit, visible banner or badge state (not just     
  implied by stale timestamps) when running on cached data — must clearly read  
  "not live" per assessment requirement.                                        
                                                                                
  6. Remove: display-mega/display-xxl hero typography, marketing copy tone,     
  full-bleed sage band as page-opening element. Keep: primary green accent,     
  rounded-xl cards, Inter body type, existing semantic color tokens             
  (positive/warning/negative) for rate deltas and status badges.                
                                                                                
  Goal: read as a data tool a user opens daily, not a landing page — density    
  and status-legibility over brand spectacle.
