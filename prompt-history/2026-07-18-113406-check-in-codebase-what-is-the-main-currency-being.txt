❯ why it's showing AED instead of MYR?                                          

  Searched for 1 pattern, called playwright 3 times, ran 2 shell commands (ctrl+o to expand)

⏺ Found it. The select's option list starts as the 30 curated codes (MYR
  included, matches fine) then gets replaced with the full 166-code API list
  once the snapshot loads. Angular's [value] binding only re-writes the DOM
  property when the bound expression itself changes — since ratesService.base()
  stays 'MYR' the whole time, Angular never re-pushes it after the option list
  is swapped out, so the native <select> just falls back to whatever the browser
  lands on after the DOM mutation (not deterministically the first option
  either). Confirmed via document.getElementById('base-select').value → wrong
  value even though 166 options loaded correctly.
